# 📈 A股交易作战系统

个人 A 股交易者的「决策作战系统」——不是行情站，而是用来**固化交易纪律、记录每一次决策、统计胜率**的工具。

> 当前进度：**模块 1 MVP**（决策框架 + 信号录入 / 命中率追踪）。
> 技术栈：React + Vite，数据存浏览器 `localStorage`（第一阶段不需要后端）。

## 功能（模块 1）

四个标签页，全部单列布局、针对窄屏 / 折叠屏优化、中文界面：

| 标签 | 作用 |
| --- | --- |
| 📝 新建决策 | 选信号等级 A/B/C → 自审清单逐项确认 → 主线三维加权打分（实时综合分）→ 保存 |
| 📋 记录看板 | 查看所有历史记录，展开看明细，**事后回填结果**（成功/失败/盈亏%）|
| 📊 命中率 | 按信号等级统计数量、命中率、平均盈亏（用回填结果计算）|
| ⚙️ 配置 | 自定义信号等级阈值 / 自审清单 / 打分维度权重——**方法论内容不写死，由你录入** |

打分默认权重：历史结构 30% + 催化/想象 30% + 资金流行为 40%（可在配置页修改）。

## 本地运行

需要 Node.js（建议 LTS）。

```bash
npm install      # 第一次安装依赖
npm run dev      # 启动开发服务器，打开提示的 http://localhost:5173
npm run build    # 生产构建
npm run lint     # 代码检查
```

## 目录结构

```
src/
  storage.js              数据层：localStorage 读写 + 默认方法论配置（默认值在这里）
  App.jsx                 主壳：标签切换 + 共享状态（配置/记录）
  index.css               全局样式（单列、窄屏/折叠屏优先）
  components/
    NewDecision.jsx       新建决策表单
    RecordList.jsx        记录看板 + 结果回填
    Stats.jsx             命中率统计
    Config.jsx            方法论自定义配置
```

## 设计要点

- **方法论可配置**：所有信号等级、清单项、打分维度只是 `storage.js` 里的*默认值*，在「配置」页可随时增删改。
- **历史记录快照**：每条记录保存当时的清单/维度文字快照，之后改配置不影响旧记录展示。
- **数据本地存储**：当前存浏览器 `localStorage`；后续阶段再迁移到后端数据库（SQLite 起步）。
