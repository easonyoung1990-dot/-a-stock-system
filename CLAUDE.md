# CLAUDE.md — 个人 A 股交易作战系统

> 这是项目总说明。每次进入本项目，请先读本文件，理解目标、技术栈、当前阶段和协作方式，再开始工作。

## 一、项目目标

把我（上海的 A 股个人交易者）已有的一套交易方法论，做成一个可交互、能记录、能长期追踪的网站。

核心定位：**这不是行情展示站**（行情看同花顺/东财即可），而是一个用来固化我的决策纪律、记录每一次决策、统计胜率的「个人交易作战系统」。

我同时是编程学习者，做这个项目也是为了系统学习前端、后端和 AI 编程。**请在每一步都用中文解释你在做什么、为什么这么做**，默认带一点教学性质，但不要拖慢节奏。所有交流用中文。

## 二、技术栈（已定）

- 前端：React + Vite
- 后端：Python + FastAPI（第三阶段才引入，现在先不用）
- 行情数据：AKShare（开源库；后端拉取 + 缓存 + 限流；属非官方数据源，仅自用、不商用）
- 数据存储：第一阶段用浏览器 localStorage；上线阶段再迁移到后端数据库（SQLite 起步）
- 部署：第四阶段处理（国内访问 + 域名备案，到时再定方案）

> **第一阶段（当前）只需要 Node.js，不需要 Python。**

## 三、四个功能模块与优先级

1. **决策框架 + 信号录入/准确率追踪 ← 现在就做这个**
2. 每日盯盘 checklist（7 个时间节点）+ 节点提醒
3. 交易日志与复盘归因
4. 自选股 / 实时行情辅助看板（依赖后端，放最后）

## 四、当前任务：模块 1 — 决策框架 + 信号录入/追踪

### 业务背景（我的方法论骨架）

我有一套结构化的入场决策方法，核心包括：

- **A/B/C 三级入场信号**：每一级有各自的硬阈值（达到阈值才算成立）
- **自审清单（self-audit checklist）**：开仓前逐项确认，防止冲动交易
- **6 层 / 9 节点分析框架**：多维度评估一笔机会
- **主线情景建模**：对候选主线做加权打分 —— 历史结构 30% + 催化/想象 30% + 资金流行为 40%

> 重要：上面这些层级、节点、阈值、清单项的**具体内容由我来录入**。第一版请把它们做成**可自定义配置**的结构（我能自己增删改每一项的定义、阈值、权重），不要把内容写死在代码里。

### 模块 1 第一版（MVP）要做到

- 「新建决策/信号」表单：选择信号等级（A/B/C）→ 按自审清单逐项确认 → 对候选主线按三项权重打分，得出综合分
- 录入后保存为一条记录（存 localStorage）
- 一个列表/看板，查看所有历史记录
- 每条记录可事后回填结果（成功/失败/盈亏）
- 一个统计页：按信号等级统计数量与命中率（用回填的结果计算）

### 开发原则

- **先做能跑起来的最小版本，再逐步迭代**，不要一次堆太多功能
- 每完成一步，告诉我如何在本地看到效果
- 方法论内容、配置项都可由我编辑，不写死在代码里

## 五、UI / 交互规范（重要）

- **单列布局**，不要 2–3 列网格；按钮、标签页垂直堆叠
- 针对 **OPPO Find N 折叠屏**优化（展开后约半屏宽的窄屏）；表格/图表 max-width:100%，允许横向滚动
- 响应式：flex-wrap / grid auto-fit；顶部标签在窄屏转为横向滚动或下拉
- 中文界面
